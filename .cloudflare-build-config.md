# Cloudflare Pages Build Configuration

## Настройки сборки в Cloudflare Pages UI

При создании проекта в Cloudflare Pages укажите следующие настройки:

### Framework preset
**Vite** (или **None**)

### Build command
```bash
cd frontend && npm install && npm run build
```

### Build output directory
```
frontend/dist
```

### Root directory (optional)
Оставьте пустым или укажите `/` (корень репозитория)

### Environment variables (если нужны)
- `VITE_API_URL` - URL вашего backend API

---

## Важно!

Cloudflare Pages пытается автоматически определить тип проекта. Если он видит `requirements.txt` в корне, он может попытаться установить Python зависимости.

**Решение:** Убедитесь, что в настройках Build указаны правильные команды выше, которые явно указывают на `frontend` директорию.

---

## Альтернатива: Использование wrangler.toml

Если хотите использовать конфигурационный файл, создайте `wrangler.toml` в корне проекта:

```toml
[env.production]
compatibility_date = "2024-01-01"

[build]
command = "cd frontend && npm install && npm run build"
cwd = "frontend"
```

Но Cloudflare Pages обычно использует настройки из UI, а не из `wrangler.toml`.
