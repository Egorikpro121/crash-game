# Пошаговая инструкция: Куда нажимать в Cloudflare

## Текущая ситуация
✅ Проект `crash-game` создан  
❌ Сборка падает: "Latest build failed"

---

## Шаг 1: Откройте проект

1. **Нажмите на название проекта** `crash-game` в списке
2. Вы попадете на страницу проекта

---

## Шаг 2: Посмотрите логи ошибки

1. В проекте перейдите на вкладку **"Deployments"** (сверху)
2. Найдите последний deployment с красным статусом ❌
3. **Нажмите на него**
4. Прокрутите вниз до секции **"Build log"**
5. **Скопируйте последние строки с ошибкой** и отправьте мне

---

## Шаг 3: Исправьте настройки сборки

1. В проекте перейдите на вкладку **"Settings"** (сверху)
2. В левом меню выберите **"Builds & deployments"**
3. Найдите секцию **"Build configuration"**
4. Нажмите кнопку **"Edit"** или **"Configure build"**

### Укажите следующие настройки:

**Build command:**
```
cd frontend && npm install && npm run build
```

**Build output directory:**
```
frontend/dist
```

**Root directory:**
```
/ (оставьте пустым или укажите просто "/")
```

5. Нажмите **"Save"**

---

## Шаг 4: Пересоберите проект

1. Вернитесь на вкладку **"Deployments"**
2. Найдите последний failed deployment
3. Нажмите на **три точки** (⋮) справа от него
4. Выберите **"Retry deployment"** или **"Retry build"**

Или просто сделайте новый коммит в GitHub - Cloudflare автоматически пересоберет проект.

---

## Важно проверить!

### Это Pages или Worker?

- Если в URL видно `/pages/` → это **Pages** ✅ (правильно)
- Если в URL видно `/workers/` → это **Worker** ❌ (неправильно)

Если это Worker:
1. Удалите этот проект
2. Создайте новый **Pages** проект
3. Подключите репозиторий заново

---

## После успешной сборки

Вы получите домен вида: `crash-game-xxxxx.pages.dev`

Этот домен нужно будет указать в настройках Telegram Bot.

---

Отправьте мне:

2. Последние строки из "Build log" с ошибкой

И я помогу исправить!
